github.com/aosp-mirror

O mesmo Android que você conhece e gosta.

A conta do GitHub aosp-mirror fornece um espelho somente leitura de alguns dos repositórios mais comuns do Projeto Android de código aberto. Você pode visualizar o código-fonte e revisar o histórico do projeto, assim como qualquer outro repositório Git. Porém, por se tratar de um espelho, algumas coisas como contribuir com o projeto funcionam de forma um pouco diferente, sobre o que falaremos a seguir.

Envio de bugs e patches

Android tem processos específicos para reportar bugs e enviar patches que os desenvolvedores precisarão seguir para contribuir. A equipe do Android não monitora nenhum problema ou pull request registrado nos mirrors do GitHub. Informações adicionais podem ser encontradas na seção de contribuição do website do Android.

Usando a ferramenta de repositório

Ao trabalhar com o código-fonte do Android, os desenvolvedores usam a ferramenta de repositório para verificar o código dos repositórios git oficiais. Devido às diferenças em como os repositórios são nomeados, o repo não funcionará com o espelho do Android no Git. Várias pessoas publicaram guias sobre usando repo com Git, mas você está sozinho lá.

Melhor esforço

Por fim, a atualização dos espelhos do Git é fornecida com base no "melhor esforço" (embora geralmente esteja menos de uma hora atrasada). Além disso, nem todos os repositórios podem ser espelhados no Git. Para ver a lista mais atualizada de repositórios Android, visite https://android.googlesource.com/.
